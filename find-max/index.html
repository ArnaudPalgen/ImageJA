Find 3D Max

Author: Wayne Rasband (wsr at nih.gov)
History: 2010/12/01: First version
Source: Find_3D_Max.java
Installation: Drag and drop Find_3D_Max.java onto the "ImageJ" window and compile and run it by pressing ctrl-r (File>Compile and Run).
Description: I developed this plugin to compare the speed of various data type independent pixel access methods. What it does is find the first pixel in an image, stack or hyperstack that contains the maximum value. It then writes to the Log window the maximum value, the location of the pixel and the processing rate in million of pixels per second. In addition, it creates a point selection at the location of the pixel and positions the stack or hyperstack so that the image containing the pixel is displayed.

For comparision, Imglib (Imglib_Find_Max.java), Pixlib (Pixlib_Find_Max.java) and raw byte array (Byte_Find_Max.java) versions of this plugin are also available. The Imglib version requires Fiji and the Pixlib version requires Dimiter Prodanov's pixlib.jar library.

This plugin runs up to seven times faster on 64-bit versions of Java because the JIT compiler is able to inline the getf() method. For best performance, run the plugin by selecting it from the Plugins menu. Run it repeatedly by pressing shift-r (Process>Repeat Command) for even better performance.
